# API Reference

## REST Endpoints

### ESP32 Endpoints

These are called by the firmware, not from the browser.

---

#### `POST /api/esp/register`

Called on ESP32 boot after WiFi connects. Upserts the device record. If `factoryReset: true`, deletes all existing relays and clears `homeId`.

**Request body:**

```json
{
	"apiKey": "ehk_...",
	"macAddress": "AA:BB:CC:DD:EE:FF",
	"name": "Living Room ESP32",
	"ssid": "HomeWiFi",
	"firmwareVersion": "1.2.0",
	"factoryReset": false
}
```

**Response `200`:**

```json
{
	"deviceId": "...",
	"relays": [{ "id": "...", "pin": 5, "label": "Light", "state": false }]
}
```

**Errors:** `401` bad/inactive API key, `500` DB error.

---

#### `POST /api/esp/heartbeat`

Called every 60 seconds by the ESP32 as a fallback sync mechanism. ESP32 reports physical relay states (authoritative); server returns desired states (picks up any missed commands).

`lastSeenAt` update is throttled - only written to DB if >30s since last write.

**Request body:**

```json
{
	"deviceId": "...",
	"apiKey": "ehk_...",
	"relayStates": [{ "id": "...", "state": true }]
}
```

**Response `200`:**

```json
{
	"relays": [{ "id": "...", "pin": 5, "state": false }]
}
```

**Errors:** `400` missing fields, `401` bad API key or device/key mismatch.

---

### Auth Endpoints

All auth endpoints are under `/api/auth/`.

#### `POST /api/device/[id]/firmware`

Upload a firmware `.bin` for OTA. Requires authenticated session (owner only). `Content-Type: multipart/form-data` with a `firmware` field.

**Response `200`:** `{ ok: true, size: <bytes> }`  
**Errors:** `401` unauthorized, `403` not owner, `400` missing/wrong file type.

---

#### `GET /api/device/[id]/firmware?token=<token>`

Download the uploaded firmware binary. Used by ESP32 during OTA; `token` is a one-time 10-minute token generated by the WS server when `triggerOta` is called. Token is consumed on first use.

**Response `200`:** binary `application/octet-stream`  
**Errors:** `401` missing/invalid token, `404` no firmware uploaded, `503` WS server unreachable.

---

#### `POST /api/auth/register`

Creates a new user account. Hashes password with bcrypt (12 rounds). Generates 24h email verification token and sends verification email via Brevo.

**Body:** `{ email, password, name }`  
**Response:** `{ ok: true }` or `{ error: "..." }`

---

#### `GET /api/auth/verify-email?token=&email=`

Validates the email verification token, sets `emailVerified = now()`, deletes the token. Redirects to `/auth/login?verified=true`.

---

#### `POST /api/auth/resend-verification`

Replaces any existing verification token with a fresh 24h token and resends the email.

**Body:** `{ email }`

---

#### `POST /api/auth/check-verified`

Polls whether a credential user's email is verified. Only returns `false` for credential users - prevents OAuth account enumeration.

**Body:** `{ email }`  
**Response:** `{ verified: boolean }`

---

#### `POST /api/auth/forgot-password`

Generates a 1-hour password reset token and sends a reset email. Does **not** reveal whether the email address exists in the system.

**Body:** `{ email }`

---

#### `POST /api/auth/reset-password`

Validates the reset token, bcrypt-hashes the new password (12 rounds), deletes the token.

**Body:** `{ token, email, password }`

---

#### `GET/POST /api/auth/[...nextauth]`

NextAuth v5 catch-all handler. Handles OAuth callbacks, sign-in, sign-out, session reads.

---

### User Endpoints

Require an authenticated session.

#### `POST /api/user/update-name`

Updates the current user's display name.

**Body:** `{ name: string }`

#### `POST /api/user/change-password`

Verifies current password with `bcrypt.compare`, then sets new password hash.

**Body:** `{ currentPassword, newPassword }`

## tRPC Procedures

Base path: `/api/trpc`. All procedures except `user.updateSelf` require an authenticated session (via `protectedProcedure`). Use the `api.*` client in React components.

### `device`

| Procedure            | Input                                      | Notes                                                                 |
| -------------------- | ------------------------------------------ | --------------------------------------------------------------------- |
| `list`               | -                                          | All devices the user owns (via API keys)                              |
| `get`                | `{ id }`                                   | Returns device + relays + switches + `accessLevel: "owner"\|"shared"` |
| `update`             | `{ id, name?, notes? }`                    | Owner only                                                            |
| `delete`             | `{ id }`                                   | Owner only                                                            |
| `pingDevice`         | `{ deviceId }`                             | Calls WS `/ping-device`; updates `lastSeenAt` if online               |
| `toggleRelay`        | `{ relayId, state }`                       | Checks relay access; calls `/push-relay`; falls back to DB if offline |
| `addRelay`           | `{ deviceId, pin, label, icon }`           | Owner only; max 8 per device; rejects pins 34–39                      |
| `updateRelay`        | `{ relayId, label?, icon?, pin?, order? }` | Owner only; rejects pins 34–39                                        |
| `deleteRelay`        | `{ relayId }`                              | Owner only                                                            |
| `addWifi`            | `{ deviceId, ssid, password }`             | Owner only; max 4 extra networks; pushes `wifi_config` to device      |
| `removeWifi`         | `{ deviceId, index }`                      | Owner only; removes by index; pushes updated list to device           |
| `updateServerConfig` | `{ deviceId, host, port, tls }`            | Owner only; persists in DB + pushes `server_config` to device         |
| `triggerOta`         | `{ deviceId }`                             | Owner only; device must be online; firmware must be uploaded first    |

### `home`

| Procedure           | Input                        | Notes                                                 |
| ------------------- | ---------------------------- | ----------------------------------------------------- |
| `list`              | -                            | All owned homes with room/device counts               |
| `get`               | `{ id }`                     | Home + rooms (with relays+devices) + devices + shares |
| `create`            | `{ name }`                   | -                                                     |
| `update`            | `{ id, name }`               | Owner only                                            |
| `delete`            | `{ id }`                     | Owner only; must have no devices                      |
| `assignDevice`      | `{ deviceId, homeId\|null }` | `null` unassigns; also unassigns relays from rooms    |
| `unassignedDevices` | -                            | Devices with no home (or orphaned homeId)             |

### `room`

| Procedure          | Input                   | Notes                                               |
| ------------------ | ----------------------- | --------------------------------------------------- |
| `list`             | `{ homeId }`            | Requires owner or homeShare access                  |
| `get`              | `{ id }`                | Returns room + relays + `accessLevel`               |
| `create`           | `{ homeId, name }`      | Owner only                                          |
| `update`           | `{ id, name?, order? }` | Owner only                                          |
| `delete`           | `{ id }`                | Owner only; relays become unassigned (not deleted)  |
| `assignRelay`      | `{ relayId, roomId }`   | Relay's device must be in the same home as the room |
| `unassignRelay`    | `{ relayId }`           | Sets `roomId = null`                                |
| `unassignedRelays` | `{ homeId }`            | Relays in this home with no room assignment         |

### `sharing`

All home-level procedures require being the home owner. Room procedures require owning the home. Relay procedures require owning the device.

| Procedure          | Input                 | Notes                                                                     |
| ------------------ | --------------------- | ------------------------------------------------------------------------- |
| `shareHome`        | `{ homeId, email }`   | Grants access to all rooms + relays; refreshes all device subscriber sets |
| `unshareHome`      | `{ homeId, userId }`  | -                                                                         |
| `listHomeShares`   | `{ homeId }`          | -                                                                         |
| `shareRoom`        | `{ roomId, email }`   | Grants access to all relays in room                                       |
| `unshareRoom`      | `{ roomId, userId }`  | -                                                                         |
| `listRoomShares`   | `{ roomId }`          | -                                                                         |
| `shareRelay`       | `{ relayId, email }`  | Grants access to a single relay                                           |
| `unshareRelay`     | `{ relayId, userId }` | -                                                                         |
| `listRelayShares`  | `{ relayId }`         | -                                                                         |
| `listSharedWithMe` | -                     | Returns `{ homes[], rooms[], relays[] }` shared with current user         |

After any share/unshare, calls `/refresh-device-subscribers` on affected devices to rebuild the WS fan-out subscriber set.

---

### `schedule`

All procedures require owning the relay's device. Max 10 schedules per relay.

| Procedure | Input                                                              | Notes                                |
| --------- | ------------------------------------------------------------------ | ------------------------------------ |
| `list`    | `{ relayId }`                                                      | -                                    |
| `create`  | `{ relayId, label, hour, minute, daysOfWeek[], action, timezone }` | `action`: `true` = ON, `false` = OFF |
| `update`  | `{ id, ...fields }`                                                | Any subset of create fields          |
| `delete`  | `{ id }`                                                           | -                                    |
| `toggle`  | `{ id, enabled }`                                                  | Enable/disable without deleting      |

**Schedule fields:**

- `hour`: 0–23
- `minute`: 0–59
- `daysOfWeek`: array of integers 0 (Sun) through 6 (Sat)
- `timezone`: IANA timezone string (default `"Asia/Kolkata"`)

---

### `switch`

| Procedure       | Input                                                 | Notes                                                           |
| --------------- | ----------------------------------------------------- | --------------------------------------------------------------- |
| `list`          | `{ deviceId }`                                        | Requires device access                                          |
| `listAllRelays` | -                                                     | All user's relays across all devices (for cross-device linking) |
| `add`           | `{ deviceId, pin, label, switchType, linkedRelayId }` | Validates relay belongs to same user; calls `/push-switch-add`  |
| `update`        | `{ switchId, ...fields }`                             | Calls `/push-switch-update`                                     |
| `delete`        | `{ switchId }`                                        | Calls `/push-switch-delete`                                     |

`switchType`: `"two_way"` | `"three_way"` | `"momentary"`
